# VoltStock - Brand Guidelines

**Dominio:** voltstock.pt
**Mercado:** Portugal
**Sector:** E-commerce de Electronica Premium
**Ultima actualizacao:** Abril 2026

---

## 1. Essencia da Marca

### Missao

Tornar o hardware premium acessivel a todos os portugueses, com uma experiencia de compra moderna, transparente e sem complicacoes.

### Visao

Ser a referencia numero um em Portugal para compra de electronica online — reconhecida pela qualidade, design e servico.

### Valores

| Valor | Significado na Pratica |
|-------|----------------------|
| **Transparencia** | Precos claros, sem custos escondidos, reviews honestas |
| **Qualidade** | Produtos de marcas premium, experiencia de compra impecavel |
| **Acessibilidade** | Precos competitivos, linguagem tecnica mas compreensivel |
| **Inovacao** | Design moderno, tech-forward, sempre actualizado |
| **Confianca** | Garantia, suporte real, politica de devolucao justa |

### Posicionamento

"A VoltStock e a loja de electronica premium em Portugal que combina um catalogo curado de hardware de qualidade com precos justos, envio rapido e uma experiencia de compra premium."

**Nao somos:**
- Um marketplace generico (tipo Amazon/AliExpress)
- Uma loja de electromesticos gerais
- Uma loja barata de hardware duvidoso
- Uma empresa sem cara (somos portugueses, para portugueses)

## 2. Voz e Tom da Marca

### Personalidade da Marca

| Tracos | O que significa |
|--------|----------------|
| **Tecnico mas acessivel** | Sabemos do que falamos, mas explicamos de forma simples |
| **Confiavel** | Nunca exageramos, nunca prometemos demais |
| **Premium sem ser elitista** | Qualidade alta, mas linguagem descontraida |
| **Portugues** | Falamos como portugueses, com expressoes e humor local |
| **Apaixonado** | Genuinamente entusiastas por tech — nao e corporativo |

### Tom por Contexto

| Contexto | Tom | Exemplo |
|----------|-----|---------|
| Redes sociais | Descontraido, entusiasta, directo | "Esta GPU? Uma maquina. Stock limitado, nao percas." |
| Email marketing | Amigavel, informativo, pessoal | "Ola Miguel, temos novidades que vais adorar." |
| Pagina de produto | Tecnico, objectivo, completo | "12GB GDDR6X, 7680 CUDA cores, TDP 220W. Ray tracing e DLSS 3.5." |
| Suporte ao cliente | Empatico, claro, resolutivo | "Compreendo a situacao. Vamos resolver isto hoje." |
| Blog | Educativo, acessivel, detalhado | "Vamos explicar passo a passo como escolher a GPU certa para ti." |
| Publicidade | Directo, impactante, com urgencia | "Hardware premium. Precos justos. Envio rapido. Descobre." |
| Comunicacao de crise | Transparente, honesto, proactivo | "Tivemos um atraso nos envios. Eis o que estamos a fazer para resolver." |

### Guia de Escrita

**FAZER:**
- Usar "tu" (nunca "voce" — demasiado formal para o nosso publico)
- Escrever frases curtas e directas
- Usar numeros concretos ("entrega em 1-3 dias", nao "entrega rapida")
- Incluir especificacoes tecnicas relevantes
- Ser honesto sobre limitacoes de produtos
- Usar humor tech quando apropriado

**NAO FAZER:**
- Usar jargao desnecessario sem explicacao
- Escrever blocos de texto enormes sem estrutura
- Prometer prazos que nao podemos cumprir
- Usar superlatives vazios ("o melhor do mundo", "incrivel", "fantastico")
- Copiar descricoes do fabricante sem adaptar
- Usar ingles quando existe equivalente portugues natural

### Vocabulario da Marca

| Preferimos | Em vez de |
|-----------|-----------|
| Setup | Configuracao |
| Upgrade | Actualizacao (quando no contexto de hardware) |
| Stock | Inventario |
| Hardware | Componentes informaticos |
| Gaming | Jogos (quando em contexto de perifericos/PC) |
| Build | Montagem (quando em contexto de PC) |
| Review | Analise/critica |
| Envio | Expedicao |
| Suporte | Assistencia tecnica |

**Nota:** Palavras inglesas comuns no mundo tech sao aceitaveis e ate preferidas quando o publico portugues as usa naturalmente (setup, gaming, upgrade, build). Nao forcar traducao quando soa artificial.

### Gradientes

| Nome | Definicao | Uso |
|------|-----------|-----|
| **Gradiente Hero** | #3B82F6 → #8B5CF6 | Hero banners, CTAs premium |
| **Gradiente Card** | #141414 → #1E1E1E | Cards de produto com hover |
| **Gradiente Glassmorphism** | rgba(255,255,255,0.05) → rgba(255,255,255,0.10) | Overlays, modais, tooltips |

### Regras de Cor

1. **Fundo sempre escuro** — #0A0A0A ou #141414. Nunca branco como fundo principal.
2. **Texto sempre claro** — #FFFFFF para primario, #E5E5E5 para secundario, #999999 para terciario.
3. **Azul para accao** — Botoes, links, elementos interactivos.
4. **Violeta para premium** — Badges, destaques de produto, ofertas especiais.
5. **Cores de estado** — Verde (sucesso), Amarelo (atencao), Vermelho (erro) — consistentes em toda a plataforma.
6. **Contraste minimo WCAG AA** — 4.5:1 para texto normal, 3:1 para texto grande.

## 4. Tipografia

### Fonts Primarias

| Uso | Font | Peso | Tamanho | Fallback |
|-----|------|------|---------|----------|
| **Headlines H1** | Inter | Bold (700) | 48-64px (desktop) / 32-40px (mobile) | system-ui, sans-serif |
| **Headlines H2** | Inter | SemiBold (600) | 36-48px / 24-32px | system-ui, sans-serif |
| **Headlines H3** | Inter | SemiBold (600) | 24-32px / 20-24px | system-ui, sans-serif |
| **Body** | Inter | Regular (400) | 16-18px / 14-16px | system-ui, sans-serif |
| **Body Small** | Inter | Regular (400) | 14px / 12px | system-ui, sans-serif |
| **Precos** | Inter | Bold (700) | 24-32px (destaque) / 18-20px (lista) | system-ui, sans-serif |
| **Badges/Tags** | Inter | Medium (500) | 12-14px | system-ui, sans-serif |
| **Codigo/Specs** | JetBrains Mono | Regular (400) | 14-16px | monospace |

### Regras Tipograficas

1. **Inter** e a font principal para tudo — consistente, legivel, tech-friendly.
2. **JetBrains Mono** apenas para especificacoes tecnicas, codigos de desconto e dados tabulares.
3. **Line-height**: 1.5 para body, 1.2 para headlines.
4. **Letter-spacing**: -0.02em para headlines, 0 para body.
5. **Nunca usar** serif fonts, script fonts, ou fonts decorativas.
6. **Maximo 3 pesos** por pagina (Regular, SemiBold, Bold).
7. **Tamanho minimo** de texto: 12px (nunca mais pequeno).

### Hierarquia Visual

```
H1: Titulo da pagina (1 por pagina)
  H2: Seccoes principais
    H3: Subseccoes
      Body: Texto corrido
        Body Small: Notas, disclaimers, metadata
          Badge: Tags, categorias, estados
```

## 5. Guia de Imagens

### Fotografia de Produto

**Estilo:**
- Fundo escuro (#0A0A0A ou #141414) — nunca branco
- Iluminacao dramatica lateral (estilo editorial tech)
- Produto centrado, angulo ligeiramente elevado (3/4 view)
- Sombra suave difusa (nao dura)
- Resolucao minima: 1200x1200px para catalogo, 2400x1600px para hero

**Angulos obrigatorios por produto:**
1. Vista frontal (3/4) — imagem principal
2. Vista lateral
3. Detalhe / Close-up de feature chave
4. Produto em contexto (no setup, em uso)
5. Embalagem (opcional mas recomendado)

**Proibido:**
- Fundos brancos genericos (demasiado Amazon)
- Fotos com flash directo (aspecto amador)
- Imagens de baixa resolucao ou pixelizadas
- Marcas de agua de stock photos
- Imagens com texto sobreposto (excepto banners de marketing)

### Banners e Graficos

**Hero banners:**
- 1920x800px (desktop) / 1080x1920px (mobile)
- Produto em destaque no lado direito
- Texto no lado esquerdo (headline + subheadline + CTA)
- Gradiente de fundo #0A0A0A → transparente
- Brilho neon subtil atras do produto

**Cards de produto:**
- 400x400px ou 600x600px
- Hover effect: escala 1.03 + sombra azul subtil
- Badge no canto (Novo, -X%, Esgotado)

**Social media:**
- Instagram Feed: 1080x1080px (quadrado)
- Instagram Story/Reel: 1080x1920px (9:16)
- Facebook: 1200x628px
- TikTok: 1080x1920px (9:16)

### Iconografia

- Estilo: Line icons, stroke 1.5-2px
- Cor: #FFFFFF (sobre fundo escuro) ou #3B82F6 (destaque)
- Tamanho: 20-24px inline, 32-48px como feature icons
- Set recomendado: Lucide Icons (open source, consistente com Inter)
- Nunca misturar estilos de icon (filled + outlined)

## 6. Elementos de Design

### Glassmorphism (Identidade Visual Core)

```css
/* Card glassmorphism padrao */
background: rgba(255, 255, 255, 0.05);
backdrop-filter: blur(12px);
border: 1px solid rgba(255, 255, 255, 0.10);
border-radius: 16px;
```

**Regras:**
- Usar em cards, modais, tooltips e overlays
- Nunca em backgrounds de pagina inteira
- Opacidade do fundo entre 0.03 e 0.10
- Blur entre 8px e 16px
- Border sempre visivel (mesmo que subtil)

### Espacamento

| Token | Valor | Uso |
|-------|-------|-----|
| xs | 4px | Espacamento entre icone e texto |
| sm | 8px | Espacamento intra-componente |
| md | 16px | Espacamento entre elementos |
| lg | 24px | Espacamento entre seccoes |
| xl | 32px | Espacamento entre blocos |
| 2xl | 48px | Espacamento entre seccoes de pagina |
| 3xl | 64px | Margem de seccao hero |
| 4xl | 96px | Espacamento vertical maior entre seccoes |

### Border Radius

| Uso | Valor |
|-----|-------|
| Botoes | 8px |
| Cards | 16px |
| Inputs | 8px |
| Badges | 9999px (pill) |
| Imagens de produto | 12px |
| Modais | 24px |

### Animacoes (Framer Motion)

**Principios:**
- Suaves e premium (nunca bruscas ou exageradas)
- Duration: 0.2-0.5s para micro-interacoes, 0.5-1s para transicoes de pagina
- Easing: easeOut para entradas, easeIn para saidas, spring para interacoes
- Stagger: 0.05-0.1s entre items em listas

**Animacoes padrao:**
- Fade in: opacity 0→1, translateY 20px→0
- Card hover: scale 1→1.03, shadow increase
- Page transition: opacity + translateX
- Skeleton loading: pulse animation em #1E1E1E → #2A2A2A

## 7. Comunicacao - Dos e Don'ts

### DO (Fazer Sempre)

| Area | Regra |
|------|-------|
| Linguagem | Usar "tu", nunca "voce" |
| Linguagem | Escrever em PT-PT nativo (nao traduzir de EN ou PT-BR) |
| Linguagem | Ser especifico com numeros e specs |
| Linguagem | Usar termos tech que o publico conhece (setup, gaming, upgrade) |
| Visual | Manter dark mode consistente |
| Visual | Usar hierarquia visual clara (tamanhos, pesos, cores) |
| Visual | Glassmorphism em cards e overlays |
| Suporte | Responder em menos de 24h (meta: 4h) |
| Suporte | Usar o nome do cliente |
| Suporte | Ser honesto sobre prazos e limitacoes |
| Marketing | Mostrar precos reais (nunca "desde", nunca sem IVA) |
| Marketing | Incluir specs completas nos produtos |
| Marketing | Usar fotos de produto de qualidade |

### DON'T (Nunca Fazer)

| Area | Regra |
|------|-------|
| Linguagem | Nunca usar CAPS LOCK em frases inteiras (excepto codigos de desconto) |
| Linguagem | Nunca usar mais de 1 ponto de exclamacao (!) |
| Linguagem | Nunca copiar descricoes de outros sites ou fabricantes sem reescrever |
| Linguagem | Nunca usar linguagem sexista, discriminatoria ou ofensiva |
| Linguagem | Nunca prometer "o melhor preco" sem verificar |
| Visual | Nunca usar fundo branco como cor principal |
| Visual | Nunca misturar paletas de cores nao aprovadas |
| Visual | Nunca usar imagens de stock genericas de pessoas |
| Visual | Nunca usar mais de 2 fonts diferentes |
| Marketing | Nunca fazer spam de email (max 3/semana) |
| Marketing | Nunca comprar seguidores ou engagement |
| Marketing | Nunca ignorar feedback negativo — responder sempre |
| Marketing | Nunca publicar conteudo de outra marca sem credito |

## 8. Templates de Comunicacao

### Resposta a Reclamacao

```
Ola [Nome],

Lamento muito a experiencia que tiveste. [Reconhecer o problema especifico].

Vamos resolver isto: [Accao concreta que vamos tomar].

[Se aplicavel: compensacao/desconto/reenvio]

Se tiveres mais alguma duvida, estou aqui.

[Nome]
Suporte VoltStock
```

### Resposta a Review Positiva

```
Obrigado pela tua review, [Nome]! Ficamos muito contentes que
[referencia ao que elogiaram]. Conta connosco para o proximo upgrade!
```

### Resposta a Review Negativa

```
Ola [Nome], obrigado pelo teu feedback honesto. Lamentamos que
[problema]. Vamos entrar em contacto contigo directamente para resolver
esta situacao. Podias enviar-nos um email para suporte@voltstock.pt
com o numero da encomenda? Obrigado.
```

### Resposta a Pergunta Tecnica (Redes Sociais)

```
Boa pergunta! [Resposta tecnica clara e concisa]. Se precisares de
mais detalhes, temos um guia completo no nosso blog: [link]. Qualquer
duvida, estamos aqui.
```
